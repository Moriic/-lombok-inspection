Reports standard getter method that can be replaced by the lombok @Getter annotation.

Example:


class TestLombok {
    // to be replaced
    private String test;
    // don't replace,because @Getter will generate getCStartDate, not getcStartDate;
    private String cStartDate;
    // don't replace,because return has other thing
    private String noReplace;

    public String getTest() {
        return test;
    }

    public String getcStartDate() {
        return cStartDate;
    }

    public String getNoReplace() {
        return noReplace + "1";
    }
}

After the quick-fix/cleanup is applied:


@Getter
class TestLombok {
    // to be replaced
    private String test;
    // don't replace,because @Getter will generate getCStartDate, not getcStartDate;
    private String cStartDate;
    // don't replace,because return has other thing
    private String noReplace;

    public String getcStartDate() {
        return cStartDate;
    }

    public String getNoReplace() {
        return noReplace + "1";
    }
}

It only reports when the lombok library is configured. To unlombok, see the lombok site.
